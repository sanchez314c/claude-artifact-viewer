## Remaining concerns

**None critical.**

One architectural note (pre-existing, documented in code): `sandbox: false` is required because the preload script loads `dompurify`, `marked`, and `highlight.js` via `require()`. Electron's sandbox mode disables non-built-in `require()` in preload scripts. The security trade-off is mitigated by `contextIsolation: true` + `nodeIntegration: false`, which together prevent renderer-side code from accessing Node.js directly. This is a known acceptable pattern for Electron apps that need npm packages in their preload.

To fully enable sandbox in a future iteration: move all npm-dependent logic out of `preload.js` and into the renderer (using dynamic imports or bundling), then set `sandbox: true`. This would be a moderate refactor with no functional behavior change.
